## 📝 Documentation Guidelines

These docs are written to be:
- **Actionable** - Step-by-step instructions you can follow
- **Searchable** - Keywords and clear headings
- **Concise** - No unnecessary information
- **Up-to-date** - Reflects current codebase (see CHANGELOG.md for updates)

### For Confluence Migration
All markdown files are structured with:
- Clear hierarchical headings (H1-H4)
- Tables for structured data
- Code blocks with language hints
- Cross-references with relative links
- Consistent formatting
